# name-counter

Консольная утилита на Go, которая читает файл с именами (по одному на строку) и выводит количество вхождений каждого имени.

## Замечания к реализации

### Что считать именем?

Изучив задание, я намеренно не добавлял валидацию содержимого строк. Жители Санкт-Петербурга — это люди из разных стран с совершенно разными системами именования. В ряде юрисдикций имена могут содержать цифры, символы и нестандартные символы Unicode (достаточно вспомнить `X Æ A-12` — имя сына Илона Маска). Добавить валидацию значило бы неявно сузить определение «имени» — а это предположение, которого в условии задачи нет. Поэтому утилита честно считает всё, что передано.

Отдельно стоит отметить: даже если рассматривать теоретический максимум уникальных имён в мире (см. ниже), решение на основе `map` всё равно остаётся рабочим — так что ужесточать входные данные нет никакой необходимости.

### Выбор структуры данных: почему `map[string]int` достаточно

Прежде чем выбрать структуру хранения, я задался вопросом: сколько уникальных имён вообще может существовать?

По данным из интернета обнаружил, что зафиксировано около **30 миллионов** уникальных имён в мире. Академические оценки лингвистов дают диапазон **5–10 миллионов** уникальных имён с учётом транслитераций и региональных вариантов.

Если взять верхнюю оценку — 30 млн записей — и допустить, что каждый ключ `map` занимает в среднем ~20 байт (строка) + 8 байт (int), то:

```
30_000_000 × 28 байт ≈ 840 МБ
```

Это предельный теоретический случай — все уникальные имена планеты в одном файле. На практике файл по Санкт-Петербургу (~5.5 млн жителей) содержит в разы меньше уникальных имён: реалистичная оценка — несколько тысяч, то есть `map` уместится в единицы мегабайт. Это полностью подходит под задачу.

Я считаю, что быть инженером — значит выбирать не идеальное в теории решение, а решение, подходящее для конкретной задачи. `map[string]int` здесь более чем достаточно.

#### Теоретическое решение для неограниченного входа

Если всё же рассматривать сценарий, где имя — произвольный набор символов и файл огромен (например, сотни миллионов строк с длинными случайными токенами), то `map` в RAM перестаёт быть пригодным. В этом случае подходит **партиционирование по хешу**:

1. Читаем файл потоком, вычисляем `hash(name) % N` для каждой строки.
2. Каждая строка попадает в один из N файлов-партиций на диске.
3. Обрабатываем каждую партицию отдельно — она уже гарантированно помещается в RAM.
4. Объединяем результаты.

Этот подход масштабируется горизонтально и не требует загрузки всего множества ключей в память одновременно.

## Ответы на вопросы задания

### 1. Что делать, если файл очень большой?

Использовать `bufio.Scanner`. Чтение идёт построчно, потоком, без загрузки всего файла в память. 
Буфер фиксированного размера (~64 КБ). Память расходуется только на `map` с уникальными именами — а их количество ограничено (см. выше).

### 2. Какие тесты написаны?

**Чтение файла и подсчёт** — чтение и подсчёт происходят за один проход, здесь важно проверить граничные случаи формата входных данных и корректность подсчёта:

| Тест | Что проверяется |
|---|---|
| Обычный файл | Базовое чтение нескольких имён |
| Подсчёт дублей | Повторяющиеся имена суммируются верно |
| Пустые строки | Игнорируются и не попадают в результат |
| Строки только из пробелов | Также игнорируются |
| Пробелы вокруг имени | Обрезаются (`  Алёна  ` → `Алёна`) |
| Пустой файл | Возвращает пустой map без ошибки |
| Нет финального `\n` | Последняя строка читается корректно |
| Регистр | `alyona` и `Alyona` — разные имена |

**Форматирование вывода** — здесь важно проверить что результат выводится в правильном формате:

| Тест | Что проверяется |
|---|---|
| Формат строки | Каждая строка имеет вид `Имя:Количество\n` |
| Порядок строк | Соответствует порядку переданного среза |
| Пустой вход | nil-срез → пустой вывод, ничего лишнего |

### 3. Можно ли выводить имена в порядке убывания частоты?

Да, это реализуемо через `sort.Slice` в `reporter.Prepare`.
В задании сортировка не указана как требование, поэтому я намеренно её не добавлял.
Порядок вывода недетерминирован (итерация по `map`) — что соответствует условию.

## Сборка и запуск

Собрать бинарник:

```bash
make build
```

Запустить:

```bash
./name-counter examples/names.txt
```

Запустить тесты:

```bash
make test
```

Собрать и сразу запустить на `names.txt`:

```bash
make run
```

### Пример

Файл `names.txt`:
```
Алёна
Миша
Алёна
Дима
```

```bash
$ ./name-counter names.txt
Алёна:2
Миша:1
Дима:1
```
